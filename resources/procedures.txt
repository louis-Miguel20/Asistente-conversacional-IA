PRUEBA TÉCNICA – Ingeniero de IA/ML (Chatbots + NLP + RAG) 

Duración sugerida: 8–12 horas (2 días máximo) 

       Objetivo General 

Construir un asistente conversacional IA capaz de responder preguntas basadas en 
información contenida en un documento elegido por el candidato, usando técnicas 
modernas de: 

•  Embeddings 

• 

Indexación vectorial 

•  RAG (Retrieval Augmented Generation) 

•  LLMs 

•  Prompt orchestration 

•  Recuperación contextual 

•  Conversación multi-turn (mínimo básico) 

El objetivo es evaluar: 

•  Nivel técnico 

•  Capacidad de aprendizaje 

•  Pensamiento arquitectónico 

•  Buenas prácticas 

•  Capacidad de implementar chatbots modernos basados en LLMs 

   Material que el candidato DEBE CONSEGUIR por su cuenta 

El candidato deberá elegir un documento técnico o de referencia, de al menos 3 a 10 
páginas, por ejemplo: 

•  Un manual de usuario 

•  Documentación técnica de una librería 

•  Una guía oficial de algún producto 

•  Capítulos de un PDF académico o técnico 

•  Documentación pública de un API 

•  Artículo técnico extenso 

•  Whitepaper 

•  Etc. 

Debe ser un documento que él mismo pueda descargar, procesar y justificar. 

   PARTE 1 — Ingesta y Procesamiento de Documento Elegido (25%) 

El candidato debe: 

✔ Seleccionar un documento técnico (PDF o HTML convertido a texto) 

Explicar en un archivo readme: 

•  Por qué lo eligió 

•  Qué tipo de preguntas puede responder 

•  Qué retos tiene el texto (ruido, tablas, imágenes) 

✔ Procesar el documento 

•  Extraer texto (PDF o scraping simple) 

•  Normalizar 

•  Limpiar 

•  Dividir en chunks (300–500 tokens) 

✔ Crear embeddings 

•  Usar cualquier modelo disponible públicamente 

o  OpenAI embeddings 

o  HuggingFace (ej. bge-large, all-MiniLM) 

o  O embeddings locales 

✔ Crear o usar una base vectorial 

•  FAISS 

•  Chroma 

•  Pinecone (free) 

•  Weaviate (local) 

Debe incluir: 

•  Código 

•  Documentación breve 

•  Explicación del pipeline 

 
   PARTE 2 — RAG: Recuperación de Contexto Inteligente (30%) 

Construir un módulo que: 

1.  Reciba una pregunta libre del usuario 

2.  Genere embeddings de la pregunta 

3.  Encuentre los chunks relevantes 

4.  Construya un prompt contextualizado 

5.  Llame un LLM (OpenAI, HuggingFace, Groq, Llama, etc.) 

6.  Devuelva una respuesta fundamentada en el documento 

✔ Debe incluir: 

•  Control básico de alucinaciones 

•  Mecanismo simple de fallback (“no tengo suficiente información”) 

✔ Debe exponer una función: 

answer_question(query: str) -> dict 

Retornando: 

{ 

  "answer": "...", 

  "context_used": ["chunk1...", "chunk2...", ...] 

} 

   PARTE 3 — API REST (25%) 

Implementar una API: 

✔ POST /ask 

Entrada: 

{ "question": "..." } 

Salida: 

{ 

  "answer": "...", 

  "context": [...] 

} 

✔ GET /health 

Solo para ver si la API corre. 

Framework permitido: 

•  FastAPI 

•  Flask 

•  Django REST Framework 

•  Node/Express con Python en backend separado (opcional) 

   PARTE 4 — Pensamiento Arquitectónico (10%) 

Debe incluir un archivo architecture.md explicando: 

•  Diagrama simple del flujo (puede ser dibujo o ASCII) 

•  Elección del modelo de embeddings 

•  Elección del modelo generativo 

•  Limitaciones del enfoque 

•  Cómo lo escalaría en producción 

•  Cómo usaría memoria conversacional 

•  Cómo integraría WhatsApp, Web o cualquier canal 

•  Cómo manejaría logs, métricas y monitoreo 

Esto es clave para diferenciar perfiles senior vs. junior. 

   PARTE 5 — Calidad + Buenas Prácticas (10%) 

Debe incluir: 

•  README claro 

•  Estructura ordenada del proyecto 

•  Commits descriptivos 

•  Código limpio
